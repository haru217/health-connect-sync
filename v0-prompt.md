# v0 Prompt: 健康管理アプリ ホーム画面リデザイン

## アプリ概要
個人向けヘルスケアアプリ「Health Connect Sync」のホーム画面。
Android端末からHealth Connect経由で健康データ（睡眠・歩数・血圧・体重・カロリー等）を収集し、4つのドメインでスコアリングしてAIキャラクターがコメントする。

## 画面の目的
ユーザーが朝アプリを開いたとき、**今日の健康状態を一目で把握**できること。

## デザイン方針
- モバイルファースト（幅390px基準）
- ダークテーマ（背景: #1a1a2e〜#16213e系）
- 角丸カード、柔らかいグラデーション
- 絵文字は使わない
- ネガティブワードは避け、色で状態を伝える

## 画面構成（上から順）

### 1. 日付ナビゲーション
- 中央に日付表示「3月3日（月）」
- 左右に前日/翌日の矢印ボタン

### 2. 総合スコア（ヒーロー要素） ← ここが一番重要
- **大きなドーナツチャート**（画面幅の40-50%）
- 円の中央にスコア数値（大きなフォント、例: 72）
- 数値の下に「今日のコンディション」等の1行テキスト
- 円の色はスコアに応じて変化:
  - 70以上: 緑系（#4ba585）
  - 50-69: 黄系（#f59e0b）
  - 50未満: 赤系（#ef4444）
- 0から実スコアまで滑らかにアニメーション

### 3. ドメイン別スコアカード（4つ）
**4つのドメインをカード形式で表示。**配置はお任せ（2x2グリッド、横スクロール、縦並び等）。

各カードの情報:
| ドメイン | スコア | 色 | 実データ | サマリー |
|---|---|---|---|---|
| 睡眠 | 65 | 黄 | 5.5h | 睡眠時間がやや短め |
| アクティビティ | 55 | 黄 | 1,834歩 | 歩数が目標の60% |
| 食事 | 82 | 緑 | 1,850kcal | カロリーバランスは良好 |
| コンディション | 78 | 緑 | 129/86 | コンディションは良好 |

各カードに含める要素:
- 小さなドーナツ or プログレスバー（スコアの視覚表現）
- ドメイン名（日本語）
- スコア数値
- 実データ値（睡眠時間、歩数等）
- 1行サマリー（省略可）
- タップ可能（対応タブに遷移）

データなしのドメインはグレーアウトで「--」表示。

### 4. 注目ポイント（AttentionSection）
黄色/赤色の警告カード。例:
- ⚠ 「睡眠時間が3日連続で6時間未満です」
- ↓ 「歩数が先週より30%減少しています」

0〜3件程度。なければ非表示。

### 5. AIキャラクターコメント
3人のAIキャラクターがそれぞれコメント。各キャラのカード:

| キャラ | 名前 | 専門 | アバター |
|---|---|---|---|
| 医師 | ユウ先生 | コンディション全般 | 丸いアバター画像 |
| 管理栄養士 | サキさん | 食事・栄養 | 丸いアバター画像 |
| トレーナー | マイコーチ | 運動・活動 | 丸いアバター画像 |

各カードに:
- 丸いアバター画像（40px程度）
- キャラ名＋肩書き
- コメント本文（80〜150文字）

コメント例:
- ユウ先生: 「今日のコンディションスコアは85点と良好ですね。春の寒暖差に備え、この調子を維持していきたいですね。」
- サキさん: 「春キャベツが美味しい時期です。ざく切りにして電子レンジで温め、ポン酢をかけるだけでも立派な一品になります。」
- マイコーチ: 「今日は1834歩と少なめですが、記録を続けているのが素晴らしいです。1時間に1回立ち上がって背伸びしてみませんか。」

## サンプルデータ
```json
{
  "date": "2026-03-03",
  "overall": { "score": 72, "color": "green" },
  "domains": {
    "sleep":     { "score": 65, "color": "yellow", "summary": "睡眠時間がやや短め" },
    "activity":  { "score": 55, "color": "yellow", "summary": "歩数が目標の60%" },
    "nutrition": { "score": 82, "color": "green",  "summary": "カロリーバランスは良好" },
    "condition": { "score": 78, "color": "green",  "summary": "コンディションは良好" }
  }
}
```

## 注意点
- スコアの視覚表現（ドーナツ等）は提案してほしいが、**総合スコアが画面の主役**であること
- ドメインカードの配置・デザインは自由に提案してOK
- 全体的に「健康アプリ」ではなく「パーソナルヘルスコーチ」の雰囲気
- 情報過多にならないよう、適度な余白を確保

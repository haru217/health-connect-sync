# Request: 初回セットアップアンケート画面

- Date: 2026-03-02
- Owner: Codex-shinsekai
- Status: `todo`
- Phase: B（ホーム画面）
- Design ref: `docs/plans/2026-03-02-health-os-design.md` §9
- Depends on: A3（ユーザープロフィールAPI）

## Background
初回アクセス時にユーザーの基本情報・目的レンズ・運動プロフィールを収集する。
回答内容はプロフィールAPIに保存し、スコア重みやAIコメントのパーソナライズに使う。

## Scope
- `web-app/src/screens/SetupScreen.tsx` 新規作成
- ルーティング追加（初回未設定時にリダイレクト）
- プロフィールAPI (`PUT /api/profile`) に保存

## 画面フロー
```
Step 1: 基本情報
  - 年齢（数値入力）
  - 性別（男性/女性/その他/回答しない）
  - 身長（cm）

Step 2: 目的レンズ選択（複数ON可）
  - 体重・体型最適化 → サブ選択: 減量/増量/維持
  - 血圧改善 → サブ選択: 目標値入力（任意）
  - 睡眠改善
  - パフォーマンス最適化

Step 3: 運動プロフィール
  - 運動頻度: ほぼなし / 週1-2 / 週3-5 / 毎日
  - 運動種別: 散歩 / ジム / ランニング / 自重 / なし
  - 運動強度: 軽め / 中程度 / 高強度OK

Step 4: 完了
  - 「設定完了！」メッセージ
  - ホーム画面へ遷移
```

## UI原則
- ステッパー形式（進捗バー表示）
- 各ステップは1画面、戻るボタンあり
- 入力は選択式メイン（タップしやすく）
- スキップ可能（後からマイページで設定変更可）
- 絵文字は使わない
- 既存デザイン言語を踏襲

## Acceptance Criteria
1. 初回アクセス時（プロフィール未設定時）にセットアップ画面が表示される
2. 4ステップのウィザード形式で入力できる
3. 回答がプロフィールAPIに保存される
4. スキップして後から設定変更が可能
5. 完了後にホーム画面へ遷移する
6. 2回目以降はセットアップをスキップしてホーム直行

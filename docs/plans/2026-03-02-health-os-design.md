## 1. プロダクト定義

### コンセプト

「複数の健康目的を横断し、安全を担保しながら、今日の最適行動を1つに統合する健康OS」

### コア仮説

日本人は世界で最も豊富なパーソナル健康データを持っている（血圧計・体組成計・スマートリング）のに、それを横断的に統合判断してくれる「かかりつけAI」がない。

### ターゲットペルソナ

- スマートリング・血圧計・体組成計を**既に使っている**、健康意識の高い日本人
- データは溜まっているが「だから何？」の答えがない人
- Health Connectに過去データがある前提（コールドスタート問題は軽減）

### 競合優位性

- Apple/Googleは汎用的かつ安全側に寄る。本アプリは日本の健康文化（血圧計・体組成計・健康診断制度）に特化
- 個人の体質に踏み込んだ判断ができる（大手はリスク回避で踏み込めない）
- 個人開発のスピードと柔軟性

### フェーズ

1. まず自分用（自分の健康データで検証）
2. 周囲の人に使ってもらう
3. 一般公開

## 2. アーキテクチャ

### 全体構造（案A: MVP延長線）

```
[Health Connect] --> [Android Sync] --> [Cloudflare Workers / D1]
                                              |
                                       [ルールエンジン]
                                       スコア算出・閾値チェック・トレンド検知
                                              |
                                       [LLMプロンプト構築]
                                       ・ルール結果
                                       ・データサマリー（過去14日分）
                                       ・ユーザープロフィール
                                       ・目的レンズ設定
                                       ・体質メモ（蓄積済み分）
                                       ・日付・季節情報
                                              |
                                       [LLM API (Claude)]
                                              |
                                       1回の呼び出しで全コメント生成
                                       ・ホーム: 3専門家統合判断
                                       ・各タブ: 担当専門家コメント
                                              |
                                       [D1にキャッシュ保存]
                                              |
                                       [React Web App] 即表示
```

### 設計判断

- ルールエンジンとLLMは直列。ルールが先に事実を整理し、LLMはその結果を踏まえてレポートを生成
- ルールエンジン結果（スコア・気づき）はLLMなしでも表示可能（コスト節約・フォールバック）
- **1日1回のLLM呼び出しで全コメントをバッチ生成**し、D1にキャッシュ。タブ切り替えは即表示
- 3専門家は1回のLLM呼び出しで同時生成（コスト抑制）

## 3. 3レイヤー構造

gpt壁打ち.mdで定義済みの構造をそのまま採用。

### 安全レイヤー（常時ON）

- 異常兆候の検知
- 慢性悪化トレンドの検知
- ベースライン逸脱の監視
- オフ不可。医療判断はしない。「警告」ではなく「早期気づき」

### 目的レンズ（並列・複数ON可）

- 体重・体型最適化（減量/増量/維持から選択）
- 血圧改善（目標値設定可）
- 睡眠改善
- パフォーマンス最適化
- ユーザーが選択する。AIが目的を上書きしない。

### 日次統合判断（出力は1つ）

- 複数目的を内部で統合し「今日の最重要1つ」を提示
- 目的は維持、戦略のみ調整
- 例: 目的=ダイエット、状態=睡眠不足+HRV低下 → 「今日は回復優先。それが結果的に脂肪減につながる」

## 5. スコアシステム

### 表示

- 円（ドーナツ/ゲージ）で表示。既存MVPのHomeScreenの円表示デザインを踏襲
- 色で状態を伝える。ネガティブワード（「低下」「注意」等）は不要
- 数字は見たい人だけ見る。パッと見の色で直感的に把握

### 色の3段階

| 色 | 意味 | スコア範囲 |
|----|------|-----------|
| 緑 | 良好 | 70以上 |
| 黄 | やや気になる | 50-69 |
| 赤 | 確認した方がいい | 50未満 |

### スコア算出ロジック（ルールベース）

| ドメイン | 入力データ | 評価基準 |
|---------|-----------|---------|
| 睡眠 | 睡眠スコア・深い睡眠時間・HRV | ベースライン比較 + 絶対値 |
| 身体 | 体重・体脂肪率・筋肉量 | 目標との距離 + 週トレンド |
| 血圧 | 収縮期/拡張期 | 医学ガイドライン閾値 + 個人トレンド |
| 活動 | 歩数・運動時間・消費カロリー | 目標達成率 + 週平均比 |

- 各ドメイン0-100。総合は加重平均
- デフォルト均等配分。目的レンズで重み変更可能
- ベースラインは過去2週間の個人平均（一般論ではなく自分比）

## 6. ルールエンジン（気づき生成）

### ルールの種類

| 種別 | 例 | トリガー |
|------|-----|---------|
| 閾値超え | 血圧が135/85を超えた | 当日データ vs 医学ガイドライン |
| トレンド | 体重が7日連続増加 | 過去N日の方向性 |
| ベースライン逸脱 | HRVが個人平均から20%以上低下 | 当日 vs 過去2週間平均 |
| 相関パターン | 睡眠スコア低い翌日は血圧が高い | 過去データのパターンマッチ |

### 出力原則

- 「警告」ではなく「気づき」。事実を優しく伝える
- ポジティブな気づきも必ず混ぜる（良い傾向は積極的に伝える）
- 気づきは最大3-5個に絞る
- この結果がそのままLLMへの入力にもなる

## 7. AIレポート（3専門家コメント）

### LLMに渡すコンテキスト

1. **ユーザープロフィール**: 年齢・性別・身長・目的レンズ・運動プロフィール・生活パターン
2. **今日のデータ**: 各ドメインのスコアと生データ
3. **ルールエンジン結果**: 気づき一覧・閾値超えフラグ
4. **過去コンテキスト**: 過去14日間のトレンドサマリー・前回の週次レポート要約・蓄積された体質メモ
5. **季節・日付情報**: 今日の日付・季節・天気（あれば）

### 3専門家のキャラ設計

絵文字は一切使用禁止。

#### ユウ先生（医師・男性）

- 漢字: 悠
- 性格: 穏やかで安心感がある。データを丁寧に噛み砕いて説明する
- 口癖: 「焦らなくて大丈夫ですよ」
- 特徴: 季節の健康ネタを自然に織り込む。権威性と安心感の両立
- 担当: ホーム総合判断 + コンディション画面（体組成・バイタル・睡眠）
- トーン例: 「血圧の傾向を見ると、睡眠の質を整えることが先決です。花粉の季節は自律神経も乱れやすいですから、まずはそこから取り組みましょう」

#### サキさん（管理栄養士・女性）

- 漢字: 咲
- 性格: 明るくて親しみやすい。食べることが好き。旬の食材をよく勧める
- 信条: 「美味しく続ける」
- 特徴: 無理な制限より楽しめる食事改善を提案。季節の食材の話題が得意
- 担当: ホーム栄養判断 + 食事画面
- トーン例: 「今の時期の新玉ねぎ、血液サラサラ効果もあるしスライスしてポン酢で食べるの最高ですよ。カリウムも摂れて血圧にも良いです」

#### マイコーチ（トレーナー・女性）

- 漢字: 舞
- 性格: ポジティブで励まし上手。小さな進歩を見逃さず褒める
- 特徴: 無理はさせないが、できることは提案する。ユーザーの運動習慣に合わせてトーンと提案内容を変える
- 担当: ホーム運動判断 + 運動画面
- 運動習慣別コメント例:
  - 運動なし: 「今日は少し歩く時間を作れると良いですね。10分の散歩でも血圧改善に効きますよ」
  - 週3ジム: 「HRV的に今日は追い込める日です。脚トレでBig3を意識してみましょう」
  - ランナー: 「疲労が抜けてきています。ペース走で心肺に刺激を入れて良いタイミングです」

### トーン共通原則

- 敬語だが堅すぎない（「～ですね」「～しましょう」）
- 否定しない（「ダメ」「やめて」は使わない）
- 理由を必ず添える（「なぜなら～」で納得感）
- データの裏付けを自然に織り込む
- 季節感・人間味を入れる（あすけんのミキちゃんのように、毎日会いたくなる人格）
- 絵文字は一切使用しない

### パーソナライズの仕組み（体質メモ）

月次レポート生成時にLLMが体質パターンを抽出しD1に保存:
- 例: 「塩分摂取と血圧の相関が強い傾向」
- 例: 「睡眠不足が2日続くと体重増加しやすい」
- 例: 「週3回以上の運動週は血圧が安定する」

次回以降の日次プロンプトに含める。使い続けるほどパーソナライズが深まる。

## 8. 時間軸レポート

| 時間軸 | フォーカス | 生成方法 | 内容 |
|--------|----------|---------|------|
| 日次 | 行動（今日どうする） | リアルタイム + 1日1回LLM | スコア + 気づき + 3専門家コメント |
| 週次 | 傾向（今週どうだった） | バッチ生成 | 週間トレンド要約・目標進捗・来週への提案・「物語」としての意味づけ |
| 月次 | 体質・構造（あなたはこういう人） | バッチ生成 | 体質傾向分析・中長期リスク評価・目標見直し提案・相関パターン発見・体質メモ更新 |

### 設計原則（gpt壁打ちから）

- フェーズは週次のみ。日次ロジックを縛らない
- フェーズは「解釈」であって「ルール」ではない
- 日次データ → 週次振り返り → フェーズ生成。フェーズは"総集編"。未来を拘束しない

## 9. 画面構成

### 変更概要

| # | 画面 | 変更内容 |
|---|------|---------|
| 1 | ホーム | 大幅強化: 総合スコア(円) + ドメイン別スコア(円x4) + 気づき + 3専門家AIレポート + レポート履歴への導線 |
| 2 | コンディション | 維持 + AIコメント追加（ユウ先生が体組成・バイタル・睡眠の各タブにコメント） |
| 3 | 運動 | 維持 + AIコメント強化（マイコーチが全期間でコメント。週間/月間にも追加） |
| 4 | 食事 | 維持 + AIコメント強化（サキさんのコメントをLLMベースに格上げ） |
| 5 | マイページ | リネーム(旧MyScreen) + 強化: プロフィール・目的レンズ・運動プロフィール・体質メモ閲覧・接続状態 |

**タブ数は5のまま維持。** レポート（週次・月次）はホーム画面のAIレポートセクションから「過去のレポートを見る」で遷移する詳細ページとして実装。既存のNoReportCard（「前回のレポートを見る」リンク）のパターンを拡張する形。

### データ項目方針

- **Health Connect準拠の表示原則**: Health Connectから取得できたデータは全て表示し、データがなければその項目は表示しない
- ユーザー属性（性別等）による事前フィルタはしない。例: 排卵日・基礎体温はデータがある人には表示され、ない人には項目自体が出ない
- これにより性別設定によるフィルタリングが不要になり、ノンバイナリーや特殊な医療ケースにも自然に対応できる
- HRVが現在コンディション画面に入っていない場合は追加が必要

### 各タブのAIコメント

- 1日1回のLLM呼び出しで全タブ分をまとめて生成
- D1にキャッシュし、タブ切り替え時は即表示
- 各タブでは担当専門家が1コメント表示
- ホーム画面のAIレポートが「統合判断」、各タブは「個別の事実+一言アドバイス」

### 初回セットアップ

```
インストール → 初回セットアップアンケート → ホーム画面
                    |- 基本情報（年齢・性別・身長）
                    |- 目的レンズ選択
                    |- 運動プロフィール（頻度・種別・強度許容）
                    |- Health Connect接続・データ取得
```

設定変更はマイページから。

### ユーザープロフィール（D1保存）

```
基本情報:
  年齢, 性別, 身長

目的レンズ:
  体重・体型最適化 (ON/OFF, 減量/増量/維持)
  血圧改善 (ON/OFF, 目標値)
  睡眠改善 (ON/OFF)
  パフォーマンス最適化 (ON/OFF)

運動プロフィール:
  運動頻度: ほぼなし / 週1-2 / 週3-5 / 毎日
  運動種別: 散歩 / ジム / ランニング / 自重 / なし
  運動強度の許容: 軽め / 中程度 / 高強度OK
```
